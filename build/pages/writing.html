Writing style

General rules

  • We use Australian English spelling and sentence case for everything, including titles and buttons.
  • User messages and user interface text should use sentence case.
  • User messages should be concise but not abrupt, comprehensive but not complicated.
  • Speak the user's language, not the computer's. Use a standard vocabulary and a natural tone, and be sure to employ contractions so sentences flow naturally.
  • If recovery or troubleshooting isn't possible, provide a message body that points the user to an entity that can help – such as a system admin or Synergetic Support
  • Clarity first. Avoid unnecessary technical information and provide useful actions the user can perform. Leave out needless words.
  • Avoid redundant or unnecessary words. When your UI text is short and direct, users can absorb it quickly and easily. Identify the most important information, express it concisely, and display it prominently so that people don’t have to read too many words to find what they’re looking for or to figure out what to do next.

Date formats

In a sentance use following format with the month truncated to three characters: 12 Oct 2011

In a grid, prefer optimise for sorting the data.
