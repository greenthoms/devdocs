Form Control Attributes

Always use the appropriate type and when appropriate use: autocorrect autocapitalize pattern novalidate and autocomplete attributes.

All forms should follow accessibility best practices:

  1. Ensure forms are logical and easy to use.
  2. Ensure forms are keyboard accessible.
  3. Associate form labels with controls.
  • Disable auto-correct and disable auto-capitalization.
  • Invoke special @ keyboard.
<div class="form-group">
    <label for="exampleInputEmail1">Email address</label>
    <input  
    type="email" 
    class="form-control" 
    id="exampleInputEmail1" 
    autocapitalize="off" 
    autocorrect="off" 
    autocomplete="email">
</div>
  • Invoke special phone keyboard.
  • Turn off autocorrect.
  • Note: iOS doesn't allow input of special characters such as parenthesis and dash with the phone keyboard.
<div class="form-group">
    <label for="phone1">Phone number</label>
    <input 
    type="tel" 
    class="form-control" 
    id="phone1"
    autocorrect="off" 
    autocomplete="tel">
</div>
  • Disable auto-correct.
  • Note: while it is recommended to use a single name field, if you split it across multiple fields, be sure to assign the appropriate autocomplete values.
<div class="form-group">
    <label for="name1">Name</label>
    <input 
    type="text"
    class="form-control"
    id="name1"
    autocorrect="off" 
    autocomplete="name">
</div>
  • Disable auto-correct.
  • Placeholder adress line number.
  • Note: be sure to update the autocomplete attribute and placeholder accordingly if using multiple address line fields.
<div class="form-group">
    <label for="address1">Address line fields</label>
    <input
    type="text" 
    class="form-control"
    id="address1"
    placeholder="Address Line 1"
    autocorrect="off">
</div>
  • Invoke numeric keyboard.
  • Where possible use a masked input.
<div class="form-group">
    <label for="creditCard1">Credit card number</label>
    <input 
    type="text"
    class="form-control"
    id="creditCard1"
    exampleInputName1
    pattern="\d*" 
    novalidate 
    autocorrect="off" 
    autocomplete="cc-number">
</div>
  • Include an access key for keyboard only usage.
<div class="form-group">
    <label for="datePicker1">Date Picker</label>
    <input 
    id="datePicker1"
    class="fullWidth"
    value="10/10/2011"
    accesskey="w" />
</div>

  • For the 'Go' button on keyboard to work, avoid using type="button".

<button type="submit" class="btn btn-primary">Submit</button>
