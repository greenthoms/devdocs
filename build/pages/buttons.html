Buttons

Primary actions

  • Reset, Cancel, and Go Back are secondary actions: Avoid using secondary actions if possible.
  • Save, Continue, and Submit are primary actions: directly responsible for task completion.
  • Ensure a clear visual distinction between primary and secondary actions with the classes btn-default and btn-primary.

Button order

  • Buttons are sorted by importance from left to right.
  • Buttons should be ordered with the primary action first in the tab order.
  • Buttons should be aligned with a formâ€™s input elements to provide a clear path to completion — this helps people complete forms faster.

Button labels

  • All button labels should be written in sentence case except for "OK" which should be uppercase.
  • Button labels should be as short as possible and clearly explain the action.

Loading state

  • Use a loading state when submitting to indicate progress.
  • Use a present participle of the orginal label. For example: Save becomes Saving...
  • Further documentation: Boostrap Stateful Button.
<button type="button" id="myButton" data-loading-text="Submitting..." class="btn btn-primary btn-lg" autocomplete="off">
  Submit
</button>

<script>
  $('#myButton').on('click', function () {
    var $btn = $(this).button('loading')
    // business logic...
    // $btn.button('reset')
  })
</script>

Disabled buttons

  • Disable primary action buttons until all mandatory fields are filled.
  • Allow users to opt out if the cannot fill all mandatory fields. (Eg Use a Cancel or Close button on modals).
<button type="button" class="btn btn-primary" disabled="disabled">Submit</button>
<button type="button" class="btn btn-default">Cancel</button>
